# ⚓ Anchor Protocol: Двузначный якорь для управления AI-ассистентами

**Anchor Protocol** — это открытый стандарт управления режимами ответа для AI-ассистентов (LLM). Один символ `⚓=XY` в конце сообщения задаёт два ключевых параметра: **краткость** и **уровень скепсиса**.

Протокол позволяет:
- ✅ Управлять объёмом ответа (от телеграфного до развёрнутого)
- ✅ Регулировать стиль (от полного согласия до жёсткого спора)
- ✅ Фиксировать верификацию (ответ не противоречит источнику)
- ✅ Создавать предсказуемый пользовательский опыт

---

## 🚀 Быстрый старт

### Для пользователя

Просто добавь `/XY` в конец своего запроса:

```

Как работает теория привязанности? /15

```

Ассистент ответит в режиме:
- **X=1** — максимально развёрнуто
- **Y=5** — сбалансированно (мнение + готовность обсуждать)

В конце ответа ты увидишь:

```

[текст ответа]
⚓=15

```

### Для разработчика

1. Реализуй парсер, извлекающий `/XY` из запроса пользователя
2. Установи режим ответа согласно таблицам
3. Добавляй `⚓=XY` в конец каждого ответа
4. Обрабатывай команду `/сброс` для возврата к ⚓=55

## 📊 Таблица режимов

### Ось X: Краткость (1–9)

| X | Режим | Объём ответа |
|---|-------|--------------|
| 1 | Максимально развёрнуто | Полное исследование, все детали |
| 2 | Очень подробно | Детальный разбор с примерами |
| 3 | Подробно | Основные идеи + ключевые примеры |
| 4 | Умеренно подробно | Суть + несколько примеров |
| 5 | Сбалансированно | Суть + ключевые аргументы (умолчание) |
| 6 | Сжато | Только суть, один-два аргумента |
| 7 | Кратко | Тезисы, детали по запросу |
| 8 | Очень кратко | 1–3 предложения, только вывод |
| 9 | Телеграфно | 1–5 слов (да/нет/число) |

### Ось Y: Скепсис (1–9)

| Y | Режим | Поведение |
|---|-------|-----------|
| 1 | Полное согласие | Без возражений, исполнительский |
| 2 | Мягкое принятие | Принимает, может подсветить риски |
| 3 | Принятие с вопросами | Уточняет, но не спорит |
| 4 | Лёгкий скепсис | Мягко сомневается, но уступает |
| 5 | Сбалансированно | Мнение + готовность изменить (умолчание) |
| 6 | Уверенный скепсис | Отстаивает позицию, но слушает |
| 7 | Скептик | Активно оспаривает, требует обоснований |
| 8 | Адвокат дьявола | Ищет слабые места |
| 9 | Жёсткий спор | Стоит до неопровержимых доказательств |

## 🎯 Примеры использования

| Запрос | Режим | Ожидаемый ответ |
|--------|-------|-----------------|
| `Сколько будет 2+2? /91` | X=9 (телеграфно), Y=1 (согласие) | `4 ⚓=91` |
| `Объясни квантовую физику /15` | X=1 (развёрнуто), Y=5 (сбалансированно) | Подробный разбор с примерами |
| `Твоя теория внимания ошибочна /59` | X=5 (сжато), Y=9 (жёсткий спор) | Аргументированный спор, кратко |
| `Как мне стать лучше? /73` | X=7 (кратко), Y=3 (мягкий) | Тезисы + уточняющие вопросы |

## 🔧 Интеграция

### Python (пример парсера)

```python
import re

def parse_anchor_mode(user_input):
    match = re.search(r'/(\d{2})$', user_input.strip())
    if match:
        mode = match.group(1)
        clean_input = user_input.replace(f'/{mode}', '').strip()
        return clean_input, int(mode[0]), int(mode[1])
    return user_input, 5, 5  # умолчание
```

API-спецификация

Рекомендуется передавать x_mode и y_mode как параметры запроса к LLM.

```json
{
  "prompt": "user message",
  "x_mode": 5,
  "y_mode": 5
}
```

## SPEC.md

# ANCHOR PROTOCOL SPECIFICATION v1.0

## 1. Назначение

Протокол определяет стандарт управления режимами ответа AI-ассистента через двузначный маркер `⚓=XY`, добавляемый в конец сообщения.

## 2. Формат

- Маркер всегда начинается с символа `⚓=`
- Далее следуют две цифры: `X` (краткость) и `Y` (скепсис)
- Диапазон: от 11 до 99
- Запрещены: нули (`⚓=00`), трёхзначные значения, отсутствие маркера

## 3. Оси

### 3.1. Ось X: Краткость (1–9)

| X | Название | Макс. объём |
|---|----------|-------------|
| 1 | Полный | не ограничен |
| 2 | Очень подробный | до 2000 слов |
| 3 | Подробный | до 1000 слов |
| 4 | Умеренный | до 500 слов |
| 5 | Сбалансированный | до 300 слов |
| 6 | Сжатый | до 150 слов |
| 7 | Краткий | до 50 слов |
| 8 | Очень краткий | до 20 слов |
| 9 | Телеграфный | до 5 слов |

### 3.2. Ось Y: Скепсис (1–9)

| Y | Название | Поведение |
|---|----------|-----------|
| 1 | Полное согласие | Не оспаривает |
| 2 | Мягкое принятие | Может подсветить риски |
| 3 | Принятие с вопросами | Задаёт уточнения |
| 4 | Лёгкий скепсис | Выражает сомнение |
| 5 | Сбалансированный | Мнение + диалог |
| 6 | Уверенный скепсис | Отстаивает позицию |
| 7 | Скептик | Активно оспаривает |
| 8 | Адвокат дьявола | Ищет слабые места |
| 9 | Жёсткий спор | Максимальная защита |

## 4. Жизненный цикл

1. **Установка:** пользователь отправляет `/XY`
2. **Фиксация:** режим сохраняется для всех последующих ответов
3. **Подтверждение:** каждый ответ содержит `⚓=XY`
4. **Смена:** новый `/XY` переопределяет режим
5. **Сброс:** команда `/сброс` возвращает к `⚓=55`

## 5. Обработка ошибок

| Ситуация | Действие |
|----------|----------|
| Невалидный код (`/100`) | Игнор, ответ с `⚓=55` |
| Отсутствие якоря в ответе | Сбой, запрос `/якорь` |
| Противоречие аксиомам | Приоритет у аксиом |

## 6. Расширяемость

Протокол допускает:
- Добавление новых осей (Z)
- Расширение диапазона (при согласованной спецификации)
- Локализацию описаний режимов
```

---

⚓=55

```
